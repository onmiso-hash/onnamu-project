1. What is RDAP?

• RDAP (Registration Data Access Protocol) is a next-generation protocol for querying internet resource registration data (RFC 7480~7484), designed by IETF to replace the aging Whois protocol.

• HTTP-based RESTful API: All data is provided in JSON format, making it highly developer-friendly.

• Bootstrap Server: This service operates its own RDAP Bootstrap Server (bootstrap.rdap.kr), serving as a hub that redirects queries to the authoritative RDAP servers worldwide.

2. API Usage Guide

✦ Our APIs can be called using standard RESTful methods. Based on https://bootstrap.rdap.kr/, use the following endpoints:

A. Bootstrap Source JSON Files

Retrieve the latest raw data synchronized from IANA.

  • Domain: https://bootstrap.rdap.kr/dns.json
  • IPv4: https://bootstrap.rdap.kr/ipv4.json
  • Object Tags: https://bootstrap.rdap.kr/object-tags.json
  • (Available for ipv6.json and autnum.json as well)

B. Real-time Redirect Service (Core Feature)

The server analyzes the input and redirects you to the authoritative RDAP server.

  • Domain Query: GET https://bootstrap.rdap.kr/domain/google.com
    → Redirects to: https://rdap.verisign.com/com/v1/domain/google.com
  • IP Address Query: GET https://bootstrap.rdap.kr/ip/8.8.8.8
    → Redirects to: https://rdap.arin.net/registry/ip/8.8.8.8 (based on range analysis)
  • AS Number Query: GET https://bootstrap.rdap.kr/autnum/15169
    → Redirects to the managing AS registry server
  • Entity Query: GET https://bootstrap.rdap.kr/entity/AS-KISA
    → Analyzes the tag and redirects to the KISA RDAP server

3. How to Test (cURL Examples)

You can quickly test via Terminal (CLI) or browser address bar.

# 1. To check the redirect URL (-v option)
curl -v https://bootstrap.rdap.kr/domain/onnamu.kr

# 2. To follow the redirect and get the final result (-L option)
curl -L https://bootstrap.rdap.kr/ip/211.231.99.17
