1. RDAP 서비스 소개

• RDAP (Registration Data Access Protocol)는 기존 Whois 서비스의 한계를 극복하기 위해 IETF에서 정의한 차세대 인터넷 주소 정보 조회 프로토콜(RFC 7480~7484 등)입니다.

• HTTP 기반 RESTful API: 모든 데이터는 JSON 형식으로 제공되어 개발자가 연동하기 매우 편리합니다.

• 부트스트랩 서버: 본 서비스는 자체 RDAP 부트스트랩 서버(bootstrap.rdap.kr)를 운영하여, 조회 대상에 따라 전 세계의 실제 RDAP 서버로 즉시 연결(Redirect)해주는 허브 역할을 수행합니다.

2. API 호출 가이드

✦ 개발된 API는 표준 RESTful 방식으로 호출할 수 있습니다. https://bootstrap.rdap.kr/를 기준으로 호출 방법은 다음과 같습니다.

A. 부트스트랩 원본 JSON 파일 호출

IANA에서 가져온 최신 데이터를 원본 그대로 받고 싶을 때 사용합니다.

  • 도메인(Domain): https://bootstrap.rdap.kr/dns.json
  • IPv4 주소: https://bootstrap.rdap.kr/ipv4.json
  • IPv6 주소: https://bootstrap.rdap.kr/ipv6.json
  • AS 번호(Autnum): https://bootstrap.rdap.kr/autnum.json
  • 엔티티 태그(Object Tags): https://bootstrap.rdap.kr/object-tags.json

B. 실시간 리다이렉트 서비스 호출 (핵심 기능)

서버가 데이터를 분석하여 해당 정보를 가진 실제 RDAP 서버 주소로 즉시 연결해줍니다.

  • 도메인 조회: GET https://bootstrap.rdap.kr/domain/google.com
    → https://rdap.verisign.com/com/v1/domain/google.com으로 리다이렉트
  • IP 주소 조회: GET https://bootstrap.rdap.kr/ip/8.8.8.8
    → https://rdap.arin.net/registry/ip/8.8.8.8으로 리다이렉트 (대역 분석 결과)
  • AS 번호 조회: GET https://bootstrap.rdap.kr/autnum/15169
    → 해당 AS를 관리하는 서버로 리다이렉트
  • 엔티티(관리자/기구) 조회: GET https://bootstrap.rdap.kr/entity/IPADM11-ARIN
    → ARIN 태그를 분석하여 북미인터넷주소관리기구(ARIN) 서버로 리다이렉트

3. 테스트 방법 (cURL 예시)

터미널(CLI) 환경에서 다음과 같이 간단히 테스트해볼 수 있습니다.

# 1. 리다이렉트 되는 주소를 확인하고 싶을 때 (-v 옵션)
curl -v https://bootstrap.rdap.kr/domain/onnamu.kr

# 2. 리다이렉트를 따라서 최종 결과까지 받고 싶을 때 (-L 옵션)
curl -L https://bootstrap.rdap.kr/ip/211.231.99.17
